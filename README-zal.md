MediaCrawler项目的时间筛选功能总结

    根据我的分析，该项目确实支持基于关键词搜索的时间筛选功能，但是不同平台的支持情
    况有所不同。以下是具体发现：

    支持时间筛选的平台：

     1. 抖音 (Douyin)：
        - 通过 PUBLISH_TIME_TYPE 配置支持时间筛选
        - 选项包括：不限（0）、一天内（1）、一周内（7）、六个月内（180）
        - 在 config/dy_config.py 中配置 PUBLISH_TIME_TYPE = 0

     2. 知乎 (Zhihu)：
        - 通过 SearchTime 枚举支持时间筛选
        - 选项包括：无时间限制（默认）、一天内、一周内、一个月内、三个月内、半年内、
          一年内
        - 客户端的 get_note_by_keyword 方法接受 search_time 参数

     3. 小红书 (Xiaohongshu)：
        - 有 SearchSortType 枚举包含 LATEST（按时间倒序）选项，可以按发布时间排序
        - 但没有在搜索API中找到直接的时间范围过滤参数

     4. 微博 (Weibo)：
        - 有实时搜索类型，但没有明确的日期范围过滤选项

    不支持时间筛选的平台：
     - 快手 (Kuaishou)
     - B站 (Bilibili)
     - 贴吧 (Tieba)

    重要注意事项：

     1. 无直接日期范围筛选：项目不支持指定确切的日期范围（例如"从2024-01-01开始"）。而
        是提供相对时间过滤（如"一周内"）。

     2. 基于配置：时间筛选通常是在平台特定的配置文件中设置，而不是通过命令行参数。

     3. 命令行支持有限：命令行界面没有直接暴露时间筛选选项，需要修改配置文件才能使用时
        间筛选功能。

    对于您提出的从2024年1月1日开始查找帖子的具体需求：

     1. 对于抖音：最长只能筛选"六个月内"（180天），无法达到2024年1月1日的要求。

     2. 对于知乎：最长可以筛选"一年内"，这取决于您运行搜索的时间，可能能够覆盖到2024年
        1月1日。

     3. 对于其他平台：没有时间筛选功能。

    该项目需要进行定制开发才能支持确切的日期范围筛选（如从2024-01-01开始），因为这
    种功能目前并未内置到系统中。

    总的来说，MediaCrawler项目确实支持基于关键词搜索的时间筛选功能，但有一定的限制
    。它提供的是相对时间筛选（如"一周内"、"一年内"），而不是绝对的日期范围（如"从20
    24-01-01"）。时间筛选功能因平台而异，抖音和知乎提供了最全面的选项，而其他平台的
    时间筛选功能有限或没有。对于您从2024年1月1日开始搜索的具体需求，需要修改项目以
    添加自定义的日期范围筛选功能，因为这一确切需求目前还不支持。



    各平台排序设置：

     1. 抖音 (DouYin)：
        - 支持三种排序方式：综合排序(0)、最多点赞(1)、最新发布(2)
        - 当前代码实现中使用的是综合排序，但在 search_info_by_keyword
          方法中已经预留了排序参数，只是在调用时未指定排序类型，因此使用默认值 SearchSortType.GENERAL

     2. 小红书 (XiaoHongShu)：
        - 支持三种排序方式：综合排序(general)、最火(popularity_descending)、最新(time_descending)
        - 当前配置文件中设置为 `popularity_descending`（最火/最多点赞排序），而不是综合排序

     3. B站 (Bilibili)：
        - 支持多种排序方式：默认("")、最多点击(click)、最新发布(pubdate)、最多弹幕(dm)、最多收藏(stow)
        - 当前搜索中使用的是 `SearchOrderType.DEFAULT`（默认排序）

     4. 微博 (Weibo)：
        - 支持多种排序方式：综合(1)、实时(61)、热门(60)、视频(64)
        - 当前实现中使用默认排序

     5. 知乎 (Zhihu)：
        - 支持多种排序方式：默认("")、最多点赞(upvoted_count)、最新(create_time)
        - 当前实现中使用默认排序

     6. 贴吧 (Tieba)：
        - 支持多种排序方式：时间倒序(1)、时间正序(0)、相关度(2)
        - 当前实现中使用的是 `SearchSortType.TIME_DESC`（时间倒序），即最新发布排序

    总结：

     - 抖音：使用综合排序
     - 小红书：使用最火/最多点赞排序（非综合排序）
     - B站：使用默认排序
     - 微博：使用默认排序
     - 知乎：使用默认排序
     - 贴吧：使用最新发布排序（非综合排序）

    所以，只有抖音和部分情况下使用综合排序，其他平台各有不同的默认排序方式。特别是贴吧已经实现了按最新发布排序，小红
    书配置为按热度排序。
